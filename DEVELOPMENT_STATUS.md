# 📌 وثيقة مرجع تطوير المشروع | Project Development Status

هذا الملف يمثل المرجع الحالي لحالة الكود والـ Business Logic المعتمد في النظام. يتم تقديمه لأدوات الذكاء الاصطناعي (AI Tools) كم سياق (Context) أساسي قبل البدء في أي تعديل أو إضافة لضمان عدم كسر الميزات الشغالة.

---

## 🛠️ البيئة والتقنيات المستخدمة (Tech Stack)
- **Backend Core:** ASP.NET Core (.NET 9.0) with PostgreSQL
- **Frontend Framework:** Angular (Reactive Forms & Reactive Controls)
- **Styling:** Tailwind CSS (Dark Mode Design System: `bg-slate-950`, `text-slate-200`)
- **State Management / Forms:** FormBuilder / FormGroup (`formControlName`)

---

## 🚀 ما تم إنجازه واستقراره بنجاح (Stable Features)
1. **تعديل حقل التاريخ (Payment Date Picker):**
   - تم تثبيت التنسيق وجعل أيقونة التقويم الافتراضية واضحة ومضيئة تماماً وتتناسب مع الخلفية الغامقة باستخدام الـ CSS الفلاتر (`filter: invert(1)` و `transform: scale(1.2)`).
   - تم ربطه بنجاح مع الـ FormControl المسمى `paymentDate`.

2. **منطق طلب العهدة (Custody Request Validation):**
   - تم بناء الـ Custom Validator المسمى `insufficientBalanceValidator` في المكون `ProjectDetailsComponent`.
   - يقوم بمقارنة المبلغ المطلوب للعهدة بالرصيد المتاح للصندوق المحدد (`availableBalance`) أو بمجموع أرصدة الصناديق للمهندس.
   - في حال تجاوز المبلغ المطلوب للرصيد المتاح، يتم إطلاق خطأ `insufficientBalance` وتعطيل زر التقديم فوراً مع إظهار رسالة تنبيه حمراء واضحة للمستخدم.

3. **لوحة تحكم أدمن الشركة (Company Admin Controls):**
   - إضافة تبويب مخصص للأدمن للتحكم ببيانات الشركة وتعديل الاسم والمنطقة والوصف ورفع الشعار والبانر وتحديثها فورياً في قاعدة البيانات.
   - إضافة مفتاح خصوصية المشروع (Show/Hide on Profile Gallery) وربطه مع عمود الخصوصية `IsPublicPortfolio` في السيرفر وتحديثه بنجاح عبر بروتوكول HTTP PUT.
   - إدارة معرض صور المشروع (Project Media Gallery):
     - تحديد حد أقصى صارم بواقع 5 صور فقط لكل مشروع.
     - فحص مساحة وحجم الصورة قبل الرفع لضمان عدم تخطيها 2MB، مع إظهار رسائل تنبيه واضحة.
     - إضافة خيار حذف الصور القديمة وربطها بطرفيات خادم الـ API عبر الـ DELETE لتنظيف مساحة التخزين.
